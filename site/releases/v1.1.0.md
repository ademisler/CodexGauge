# CodexControl 1.1.0

## Highlights

- Added Sparkle-powered macOS update checks directly in the menu bar app
- Added a release pipeline that ships both macOS and Windows artifacts
- Switched the product brand to the dark Orbit Dial mark with a green ring and red core
- Refined the website visual system and navigation iconography

## Distribution Improvements

- The macOS app now embeds `Sparkle.framework` and can check for signed updates
- Release automation can optionally sign and notarize macOS builds when Apple credentials are configured
- Windows releases now ship as a dedicated zipped package for GitHub Releases

## Product Updates

- The app menu now exposes a "Check for Updates" action
- The release website and repository docs are aligned with the new install and update flow

## Notes

- Sparkle signing keys stay local in the maintainer keychain
- Public repository assets continue to use demo-only account data
